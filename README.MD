# LLM-Parser: Характеристики печатных плат

Интеллектуальный парсер для извлечения технических характеристик печатных плат из **Excel** и **Word** файлов (в т.ч. «Лист технических требований ПП» / Technical Requirements Sheet PCB) с использованием языковых моделей (LLM) и автоматической интеграцией с Битрикс24.

## Описание

Проект использует **Mistral AI** для автоматического извлечения структурированных данных о печатных платах из неструктурированных **Excel** и **Word** файлов (в т.ч. листы технических требований ПП на русском и английском). Система интеллектуально распознает технические характеристики, преобразует их в формат Битрикс24 и автоматически отправляет в CRM через REST API.

### Как это работает

1. **Загрузка файла** → Пользователь загружает Excel (.xlsx, .xls) или Word (.docx, .doc — Word 97–2003) через веб-интерфейс
2. **Извлечение данных** → Система читает листы Excel или параграфы и таблицы из Word и преобразует в текст
3. **LLM-парсинг** → Mistral AI анализирует текст и извлекает структурированные характеристики PCB
4. **Маппинг справочников** → Текстовые значения (например, "FR4", "HASL") преобразуются в ID справочников Битрикс24:
   - **В продакшене:** ID берутся из БД (`bitrix24_dictionaries`), которая автоматически обновляется
   - **Локально:** Используются fallback-словари из кода (только для разработки)
5. **Заполнение обязательных полей** → Все 12 обязательных полей заполняются автоматически:
   - Извлеченные из Excel (материал, слои, покрытие и т.д.)
   - Значения по умолчанию для полей, которых нет в Excel (Order unit, PCB type и др.)
6. **Экспорт/Отправка** → Результаты экспортируются в CSV/XLSX/JSON или отправляются напрямую в Битрикс24

## Возможности

- 📊 **Множественные форматы экспорта** - CSV, XLSX, JSON, JSON для Битрикс24
- 🔗 **Автоматическая интеграция с Битрикс24** - Прямая отправка через REST API

## Распознаваемые характеристики

Модель `PCBCharacteristics` заполняется Mistral AI и содержит ключевые параметры печатной платы:

- **company_name** – название компании
- **board_name** – обозначение/имя платы (Identification)
- **base_material** – материал основания (Base material, например `FR4 - IT180 High Tg`)
- **board_thickness** – толщина готовой платы с допуском (Finished thickness with tolerance, mm)
- **foil_thickness** – толщина меди (CU foil thickness / Thickness CU)
- **layer_count** – количество слоев (из блока Layer PCB)
- **coverage_type** – финишное покрытие (Surface Finish, например ENIG / HASL)
- **board_size** – габариты платы (Size with tolerance, длина × ширина)
- **panelization** – параметры панели (если указаны)
- **solder_mask_colour** – цвет паяльной маски (Material, color)
- **solder_mark_colour** – цвет маркировки
- **soldering_surface** – тип монтажа / сборки
- **electrical_testing** – требования к электроконтролю
- **edge_plating** – металлизация края платы
- **contour_treatment** – обработка контура

Эти данные затем автоматически маппятся в поля смарт-процесса Bitrix24 (UF_CRM_24_*) с учётом типов полей и справочников.

### 5. Настройка справочников из БД (обязательно для продакшена)

#### Как работает система справочников

Приложение использует **двухуровневую систему** для маппинга текстовых значений в ID справочников Битрикс24:

1. **Продакшен (приоритет):** Если задана `DICTIONARIES_DB_URL`, справочники загружаются из БД
   - БД автоматически обновляется при появлении новых значений в Битрикс24
   - Используется TTL-кэш (15 минут) для быстрого доступа
   - Новые значения подхватываются автоматически без правки кода

2. **Локальная разработка (fallback):** Если БД недоступна, используются словари из `app/bitrix24_dictionaries.py`
   - Только для разработки и тестирования
   - Требует ручного обновления при изменении справочников

#### Настройка подключения к БД

**Обязательно для продакшена на сервере ФайнЛайна:**

```bash
# Формат SQLAlchemy URL зависит от типа БД:
# PostgreSQL: postgresql://user:password@host:port/dbname
# MySQL: mysql+pymysql://user:password@host:port/dbname
# MS SQL: mssql+pyodbc://user:password@host:port/dbname?driver=ODBC+Driver+17+for+SQL+Server

export DICTIONARIES_DB_URL="postgresql://user:password@localhost:5432/bitrix24_db"
```

**Опциональные настройки:**

- `DICTIONARIES_CACHE_TTL_SECONDS` (по умолчанию: 900) - Время жизни кэша в секундах
- `USE_DB_DICTIONARIES=0` - Принудительно отключить БД и использовать fallback-словари

#### Структура таблицы БД

Ожидается таблица или VIEW `bitrix24_dictionaries` со следующими столбцами:

| Столбец | Тип | Описание | Пример |
|---------|-----|----------|--------|
| `param_id` | int | ID справочника Битрикс24 (IBLOCK_ID) | 56 (для Materials) |
| `param` | text | Название справочника | "Material" |
| `item_id` | int | ID элемента справочника (для отправки в Битрикс24) | 6281 |
| `item` | text | Текстовое значение элемента | "FR4" |

**Пример данных в БД:**
```
param_id | param              | item_id | item
---------|--------------------|---------|----------
56       | Material           | 6281    | FR4
56       | Material           | 6282    | FR-4
74       | Finish Type        | 7010    | HASL
74       | Finish Type        | 7011    | ENIG
54       | No of Layers       | 6784    | 4
54       | No of Layers       | 6785    | 6
```

### 6. Настройка значений по умолчанию для обязательных полей

Некоторые обязательные поля Битрикс24 не могут быть извлечены из Excel (например, "Order unit", "PCB type"). Для них используются значения по умолчанию, которые можно настроить через переменные окружения:

```bash
# Order unit (IBLOCK_ID=50)
export BITRIX24_DEFAULT_ORDER_UNIT_ID=5256

# PCB type (IBLOCK_ID=52)
export BITRIX24_DEFAULT_PCB_TYPE_ID=6610

# Peelable SM (IBLOCK_ID=86)
export BITRIX24_DEFAULT_PEELABLE_SM_ID=6014

# Production Unit (IBLOCK_ID=160)
export BITRIX24_DEFAULT_PRODUCTION_UNIT_ID=6270
```

**⚠️ Важно:** 
- Убедитесь, что эти ID существуют в вашей БД справочников
- Если ID неверные, Битрикс24 вернет ошибку при создании элемента
- Значения по умолчанию можно изменить в `app/config.py` или через переменные окружения

### 7. Получение API ключей

#### Mistral AI API Key
1. Зарегистрируйтесь на [Mistral AI](https://mistral.ai/)
2. Перейдите в раздел API Keys
3. Создайте новый ключ и скопируйте его

#### Битрикс24 Webhook URL
1. Войдите в Битрикс24 как администратор
2. Перейдите: **Настройки → Разработчикам → Другие → Входящий вебхук**

   - Название: например, "PCB Parser"
   - Права доступа: выберите необходимые (минимум: CRM)
4. Скопируйте сгенерированный URL (формат: `https://fineline.bitrix24.ru/rest/6/<token>/crm.item.add`)


### Что происходит при отправке в Битрикс24

1. **Валидация данных** - Проверка наличия всех обязательных полей
2. **Преобразование полей** - Маппинг данных PCB в формат полей Битрикс24
3. **Поиск ID справочников** - Для каждого текстового значения:
   - Сначала проверяется кэш (TTL 15 минут)
   - Если не найдено, запрос к БД справочников
   - Если БД недоступна, используется fallback-словарь
4. **HTTP запрос** - POST к `crm.item.add` с полным payload
5. **Обработка ответа** - Извлечение ID созданного элемента или сообщения об ошибке

### Как работает маппинг справочников

1. **Извлечение текста из Excel:** LLM возвращает текстовые значения (например, "FR4", "HASL", "4 слоя")

2. **Поиск ID в справочниках:**
   ```
   Текст "FR4" → Нормализация → Поиск в БД (param_id=56) → item_id=6281
   ```

3. **Приоритет источников:**
   - **1-й приоритет:** БД справочников (если `DICTIONARIES_DB_URL` задана)
     - Кэш на 15 минут для быстрого доступа
     - Автоматическое обновление при появлении новых значений
   - **2-й приоритет:** Fallback-словари из `app/bitrix24_dictionaries.py`
     - Только для локальной разработки
     - Требует ручного обновления

4. **Алгоритм поиска:**
   - Точное совпадение (нормализованный текст)
   - Нечеткое совпадение (подстрока, общие слова)
   - Если не найдено → предупреждение в логах, поле пропускается

**Примеры маппинга:**
- `"FR4"` или `"FR-4"` → `6281` (Materials, IBLOCK_ID=56)
- `"HASL"` или `"HASL Lead-Free"` → `7010` (Finish Type, IBLOCK_ID=74)
- `"4"` или `"4 слоя"` → `6784` (No of Layers, IBLOCK_ID=54)

## Формат JSON для Битрикс24

Сгенерированный JSON файл имеет следующий формат:

```json
{
  "entityTypeId": 182,
  "fields": {
    "ufCrm24_1709799376061": "OEM-12345",
    "ufCrm24_1707768819": 7010,
    "ufCrm24_1707838248": 6281,
    "ufCrm24_1709815185": 6784,
    "ufCrm24_1708374728464": 1.6,
    "ufCrm24_1708353384301": 150.0,
```

### Типичные ошибки и их причины

#### 1. Ошибка 401 Unauthorized (Mistral AI)

**Симптомы:**
```
Mistral API вернул 401 Unauthorized. Проверьте, что переменная окружения 
`MISTRAL_API_KEY` задана и ключ действителен.
```

**Причины:**
- Отсутствует переменная окружения `MISTRAL_API_KEY`
- Неверный или истекший API ключ
- Ключ не был перезагружен после установки

**Решение:**
1. Проверьте наличие переменной:
   ```bash
   # Windows PowerShell
   echo $env:MISTRAL_API_KEY
   
   # Linux/Mac
   echo $MISTRAL_API_KEY
   ```
2. Установите/обновите ключ:
   ```bash
   export MISTRAL_API_KEY="ваш_новый_ключ"
   ```
3. **Перезапустите приложение** (переменные окружения загружаются при старте)
4. Проверьте ключ на [Mistral AI Dashboard](https://console.mistral.ai/)

#### 2. Ошибка 401 Unauthorized (Битрикс24)

**Симптомы:**
```
❌ Ошибка авторизации в Битрикс24. Проверьте webhook URL или токен.
```

**Причины:**
- Неверный формат webhook URL
- Истекший токен вебхука
- Недостаточные права доступа вебхука

**Решение:**
1. Проверьте формат URL: `https://fineline.bitrix24.ru/rest/6/<token>/crm.item.add`
2. Создайте новый вебхук в Битрикс24:
   - Настройки → Разработчикам → Другие → Входящий вебхук
   - Убедитесь, что выбраны права: **CRM** (минимум)
3. Обновите переменную окружения и перезапустите приложение

#### 3. Ошибка подключения к БД справочников

**Симптомы:**
```
ModuleNotFoundError: No module named 'sqlalchemy'
# или
OperationalError: could not connect to server
# или
Ошибка при выполнении SQL запроса
```

**Причины:**
- SQLAlchemy не установлен
- Неверный формат `DICTIONARIES_DB_URL`
- БД недоступна или неверные учетные данные
- Неверная структура таблицы `bitrix24_dictionaries`

**Решение:**

**Если SQLAlchemy не установлен:**
```bash
pip install -r requirements.txt
# или
pip install SQLAlchemy==2.0.37
```

**Если проблема с подключением:**
1. Проверьте формат URL для вашей БД:
   ```bash
   # PostgreSQL
   postgresql://user:password@host:port/dbname
   
   # MySQL
   mysql+pymysql://user:password@host:port/dbname
   
   # MS SQL Server
   mssql+pyodbc://user:password@host:port/dbname?driver=ODBC+Driver+17+for+SQL+Server
   ```
2. Проверьте доступность БД:
   ```bash
   # PostgreSQL
   psql -h host -U user -d dbname
   
   # MySQL
   mysql -h host -u user -p dbname
   ```
3. Проверьте структуру таблицы:
   ```sql
   SELECT column_name, data_type 
   FROM information_schema.columns 
   WHERE table_name = 'bitrix24_dictionaries';
   ```
4. Убедитесь, что таблица содержит данные:
   ```sql
   SELECT COUNT(*) FROM bitrix24_dictionaries;
   SELECT * FROM bitrix24_dictionaries LIMIT 5;
   ```

**Временное решение:** Отключите БД и используйте fallback-словари:
```bash
export USE_DB_DICTIONARIES=0
```

#### 4. Ошибка "Не найден ID для материала" / "Не найден ID для типа покрытия"

**Симптомы:**
```
WARNING: Не найден ID для материала: 'FR4'
WARNING: Не найден ID для количества слоев: '4'
```

**Причины:**
- Значение отсутствует в БД справочников
- Неверный `param_id` (IBLOCK_ID) в запросе
- Проблемы с нормализацией текста (регистр, пробелы, дефисы)

**Решение:**

**Если используете БД (продакшен):**
1. Проверьте наличие значения в БД:
   ```sql
   SELECT * FROM bitrix24_dictionaries 
   WHERE param_id = 56  -- для Materials
   AND LOWER(REPLACE(REPLACE(item, '-', ''), ' ', '')) LIKE '%fr4%';
   ```
2. Если значение отсутствует:
   - Добавьте его в БД (БД должна обновляться автоматически из Битрикс24)
   - Или временно добавьте в fallback-словарь
3. Проверьте правильность `param_id`:
   - Materials: `param_id = 56`
   - Finish Type: `param_id = 74`
   - No of Layers: `param_id = 54`
   - Max Copper: `param_id = 62`
4. Очистите кэш (перезапустите приложение или подождите истечения TTL)

**Если используете fallback-словари (локально):**
1. Откройте `app/bitrix24_dictionaries.py`
2. Найдите соответствующий словарь (например, `MATERIALS_DICT`)
3. Добавьте значение:
   ```python
   MATERIALS_DICT: Dict[str, int] = {
       "FR4": 6281,
       "FR-4": 6281,  # альтернативное написание
       "ВашеНовоеЗначение": реальный_ID_из_Битрикс24,
   }
   ```
4. Перезапустите приложение

5. **Проверьте webhook URL и права доступа**

## Разработка

### Структура модулей

- **`app/model.py`** - Pydantic модели для структурированного вывода LLM

3. При необходимости добавьте справочник в `app/bitrix24_dictionaries.py`
